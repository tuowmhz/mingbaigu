# AI 泡沫周期监控框架（AI Bubble Cycle Monitor）

> 这是「**AI 泡沫指数**」功能的**判断框架源文档 / roadmap**。
> 代码实现见 [`backend/app/analysis/ai_bubble.py`](../backend/app/analysis/ai_bubble.py)，前端 gauge 在 `frontend/src/App.jsx` 的 `AiBubbleGauge`。
> 由用户（产品 owner）定义的分析框架，2026-06 收录。

## 这套框架是干什么的

一个「**AI 泡沫周期监控 Agent**」：判断当前 AI 产业链股票是否进入类似 **1999–2000 互联网泡沫后期**的阶段，输出结构化风险信号、风险等级和需要跟踪的数据。

**目标不是预测顶部，而是建立纪律化的监控系统**：当泡沫后期信号逐渐增多时，提醒从"进攻模式"切换到"防守模式"。重点是识别风险**从绿→黄→橙→红的边际变化过程**，而不是猜最高点。

---

## ⚠️ 两条落地约束（实现时必须遵守）

### 1. 产品宪法：不构成投资建议
明白股宪法明确"**不构成投资建议、不做荐股榜、不提供仓位/买卖**"。因此本框架里的：
- **具体仓位百分比**（"龙头 50–70%"等）
- **个股 buy / sell / trim / 收回本金 动作**

**不作为产品输出**。它们作为**分析框架参考**保留在本文档中，但产品里只呈现：指数、信号、人话解释、风险阶段。涉及"该怎么做"的部分最多转成**教育性原则**（如"分批止盈比猜顶重要"），不针对个股、不给配置比例。

### 2. 诚实数据：拿不到就标"待接入"，绝不编
免费数据（yfinance 价格/估值/财报）只够把部分信号做实；订单 backlog、期权流、ETF 资金流、内部人交易、评级修正等需付费数据。**没接入的信号明确标注"数据待接入"、不计入评分、绝不编数凑满分**。

## 一、核心判断框架：7 个泡沫后期信号

监控的**龙头股**（高质量、有真实 AI 收入）：NVDA, AVGO, MSFT, GOOGL, AMZN, META, TSM, ASML, AMD, MU, SK Hynix, LITE, COHR
监控的**高弹性/故事股**（低收入/无盈利、强叙事）：ALAB, CRDO, NBIS, POET, IREN, CORZ, WULF, 量子计算股, AI 数据中心小盘, 光通信小盘，以及任何因 AI/CPO/硅光/HBM/量子/机器人概念暴涨但收入利润未兑现的股票。

### 信号一：龙头不再领涨，故事股开始暴涨
**定义**：AI 主线从高质量龙头领涨，转向小盘/低收入/低盈利/强叙事公司暴涨 = 泡沫后半段。
**判断标准**：
- 龙头横盘/回落，但 ALAB/CRDO/POET/量子股/小盘 AI 继续大涨 → 🟡 黄。
- 小盘故事股连续多日大幅跑赢龙头，且上涨由叙事/目标价/社媒/主题资金推动而非财报上修 → 🟠 橙。
- 龙头明显滞涨，无盈利/低收入公司连续暴涨 → 🔴 红。
**输出**：龙头指数表现、故事股篮子表现、两者相对强弱、是否"质量下沉"、风险等级。

### 信号二：好消息不涨，坏消息大跌
**定义**：泡沫中后期预期极高，好财报/好订单/好指引股价也不涨；小幅不及预期则剧烈下跌。
**监控**：财报后 1/3/5 日反应；beat/raise 后是否上涨；好消息高开低走；坏消息超额下跌；上调目标价后是否不涨反跌。
**判断标准**：
- beat & raise 但不涨 → 🟡 黄。
- 多个龙头连续"好消息不涨" → 🟠 橙。
- 轻微 miss / 保守指引导致跌 10–20% → 🔴 红。
- 龙头与高估值供应链同时出现 = "预期过满"。
**输出**：公司事件、市场预期、实际结果、股价反应、判断（正常/预期过满/泡沫后期）。

### 信号三：Capex 增长快于 monetization
**定义**：hyperscaler 持续大增 AI capex，但 AI 收入/利润/现金流/回报未同步兑现。
**监控公司**：MSFT, GOOGL, META, AMZN, ORCL, xAI, CoreWeave, Nebius, Crusoe 等。
**监控指标**：AI capex 增速、capex/revenue、capex/OCF、FCF 趋势、折旧增速、AI revenue 是否单独披露、AI revenue/AI capex、管理层对 ROI 的表述、是否债务融资支持 capex、是否 vendor financing / 长期采购承诺 / 表外风险。
**判断标准**：
- capex 快增但 AI revenue 不透明 → 🟡 黄。
- capex 增长明显压低 FCF/营业利润率 → 🟠 橙。
- 市场开始质疑 AI ROI、hyperscaler 股价对 capex 增长负面反应 → 🔴 红。
- 上游仍因 capex 扩张大涨但下游变现不清 = "上游与下游利润错配风险"。

### 信号四：供应链讲 2027/2028 故事，但当前订单放缓
**定义**：从"当前订单强劲/当季上修"转向远期 TAM、2027/2028 需求、长期路线图；若当前订单/指引/backlog/毛利放缓，但股价靠远期故事维持高估值 = 危险。
**监控**：NVDA 供应链、HBM、光通信、CPO/硅光、AI 数据中心电力、液冷、AI server、网络（Ethernet/IB/scale-up/out）。重点：NVDA, AVGO, MRVL, ALAB, CRDO, LITE, COHR, POET, TSM, ASML, MU, SK Hynix, Samsung, Dell, SMCI, VRT。
**判断标准**：
- 继续讲长期 TAM 但短期 guidance 不再上修 → 🟡 黄。
- backlog 放缓/交期缩短/客户推迟 → 🟠 橙。
- 当前收入低于预期但强调 2027/2028 → 🔴 红。

### 信号五：融资、二级发行、可转债、权证、债务融资增多
**定义**：泡沫后期利用高估值窗口融资（增发/ATM/可转债/权证/债券/vendor financing/预付款/长期采购/战略投资）。
**监控**：各类 equity/secondary/ATM/convertible/warrant/high-yield/IG debt/vendor financing/customer prepayment/strategic investment/insider selling/SBC。
**判断标准**：
- 大科技发债支持 capex：不一定坏，看 FCF 与债务关系。
- 小盘高位增发/发权证 → 🟡/🟠。
- 无盈利公司频繁融资维持扩张 → 🔴 红。
- 多个 AI 概念股密集融资 / insider selling 明显增加 → 风险上调。

### 信号六：市场从盈利增长驱动变成流动性驱动
**定义**：股价上涨不再来自 EPS/收入/订单上修，而是资金流/杠杆/期权/ETF/散户情绪/社媒/gamma squeeze/主题轮动。
**监控**：EPS/Revenue/guidance revision、price move vs estimate revision、call volume、put/call、IV、杠杆 ETF 流入、margin debt、散户成交、社媒提及、meme 行为、ETF 主题流入、空头挤压、高 beta vs 低 beta、无盈利科技指数表现。
**判断标准**：
- 涨但 EPS estimate 没上修 → 🟡 黄。
- 涨主要由 call volume / ETF inflow / 社媒推动 → 🟠 橙。
- 多只无盈利 AI 股 gamma squeeze / 散户拥挤 → 🔴 红。

### 信号七：龙头估值开始压缩，小票还在疯涨
**定义**：龙头估值先压缩、资金继续追高弹性小票——看起来仍火热，但高质量资金可能已在撤离。
**监控**：龙头 forward P/E、EV/Sales、EV/EBITDA、earnings revision；小票 P/S、涨幅、earnings revision；龙头 vs 小票相对强弱。
**判断标准**：
- 龙头估值压缩但小票估值继续扩张 → 🟡 黄。
- 龙头 EPS 上修但股价不涨，小票无盈利却大涨 → 🟠 橙。
- 龙头跌破中期趋势线，小票继续泡沫化 → 🔴 红。

## 二、总体风险评分（完整版，接入付费数据后启用）

每个信号打分：0=没出现 / 1=轻微 / 2=明确 / 3=强烈。总分 0–21。

| 总分 | 阶段 |
|---|---|
| 0–4 | 正常牛市 / 基本面驱动 |

| 9–12 | 泡沫中后期（开始分批止盈） |

## 五、分析原则

1. AI 技术真实 ≠ 可以忽略估值风险；估值高 ≠ 一定会崩。
2. 核心问题：**盈利增长速度能否快过估值压缩速度**。
3. 龙头与小盘故事股用不同标准；正股与每日 2x 杠杆产品用不同标准。
4. 看**边际变化**，不是绝对水平；不预测最高点，识别绿→黄→橙→红的过程。
5. 区分：长期核心仓 / 高弹性主题仓 / 期权彩票仓 / 杠杆交易仓。
6. 基本面仍强但风险信号增加 → "分批止盈"而非清仓；信号集中恶化 → "先降仓再观察"。

**数据缺失时明确写"该项数据缺失，无法确认"，不要编造数据。**
